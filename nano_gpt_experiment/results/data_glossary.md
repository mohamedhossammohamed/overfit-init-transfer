# Data Glossary & Sanitization Log

## Overview
During the v1.1 execution of the `nano_gpt_experiment` scaling matrix, the repository underwent a sanitization pass to adopt generic, neutral terminology for its source datasets. This glossary maps historical terminology found in completed run logs to the current, neutral terminology.

## Corpus Definitions

### Target Corpus (Shakespeare)
*   **Description:** A subset of the Tiny Shakespeare dataset, containing English-language character data.
*   **Historical Variables:** `shakespeare_text`, `shakespeare_data`
*   **Usage:** Used universally as the finetuning target for all runs, and as the pretraining dataset for "Path A" (control) runs.

### Path B Source Corpus (Arabic Text)
*   **Description:** A structured Arabic-script corpus. Chosen specifically because its character set has minimal overlap with the English target corpus.
*   **Historical Variables / Identifiers:** Any run ID, variable name, or commit message containing the string `quran` refers exclusively to the **Path B Source Corpus**. 

### Path C Source Corpus (Random Noise)
*   **Description:** An unstructured stream of pure random noise generated by uniformly sampling characters from the combined vocabulary.
*   **Historical Variables:** `random_chars`

---

## Known Limitation: Path B Corpus Discontinuity

Due to the sanitization process occurring mid-matrix execution, there is a strict discontinuity in the actual file contents used for the "Path B" pretraining dataset depending on the random seed of the run.

1.  **Seed 42 Runs (e.g., `b_2M_seed42...`)**: Trained on the original source corpus.
2.  **Seed 1337 Runs (e.g., `b_2M_seed1337...`)**: Trained on a replacement generic Arabic-script corpus (sourced from Arabic Wikipedia).

**Crucially:** The replacement corpus was byte-and-token-matched to the original corpus (exactly 631,706 uint16 train tokens, and 70,190 uint16 validation tokens). Additionally, the replacement corpus was strictly filtered to only contain characters present in the original `results/vocab.json`, ensuring the `vocab_size` (120) and overall model architectures remained identically matched across all seeds.

However, because the underlying text distribution changed, **Seed 42 and Seed 1337 are not direct replicates for Path B specifically.** Path A and Path C are unaffected and remain true replicates.
